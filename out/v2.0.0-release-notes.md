# Helio v2.0.0 — install it in one command, run it like a product

v1 proved the engine: durable journeys, a real CDP, analytics on ClickHouse, an AI copilot. v2 is about everyone else — the people who don't write code. One command installs the whole platform. A setup wizard replaces the ops checklist. Your own provider keys, your own AI, your own churn model — sealed in an encrypted vault on your own server. And billing is gone for good: **Helio is free forever, with no per-contact pricing, because the platform is yours.**

97 pull requests since v1.0.0.

## Install

**Windows (PowerShell):**

```powershell
irm https://github.com/achref-soua/helio/releases/latest/download/install.ps1 | iex
```

**macOS / Linux:**

```bash
curl -fsSL https://github.com/achref-soua/helio/releases/latest/download/install.sh | sh
```

One-time prerequisite: Docker Desktop — a normal, free app install. On Windows the installer offers to set it up for you (it asks first; allow the one restart if Windows wants it). Helio handles absolutely everything else. When it says **"Helio is up,"** open `http://localhost:3000` and the setup wizard takes it from there.

Already on an earlier v2 RC? `helio update` re-pulls images and migrates in place.

## What's new, by theme

### One command to a running platform

- The **`helio` CLI** (Windows, macOS, Linux — x64 and arm64): `install`, `update`, `status`, `doctor`, `backup`, `restore`, `uninstall`. Zero dependencies, checksum-verified, with a terminal experience that stages the install as a story (and yes, the sun banner matches the logo).
- **One-line installers** for every OS; the Windows one even offers to install Docker Desktop via winget when it's missing.
- **First-run setup wizard**: name, email, password, organization — one screen, and you're the owner. Optional sample data fills the workspace with realistic contacts, campaigns, journeys, charts, and behavioral history so nothing looks empty while you explore.
- **Backups built in**: a backup sidecar runs `pg_dump` on schedule, the admin panel triggers and tracks runs ("Run now" answers instantly), and `helio restore` walks recovery.
- **Install Helio as an app** (PWA) on desktop and mobile; the dashboard is touch-friendly throughout.
- One install serves a whole team — point teammates at the same `APP_URL` and invite them.

### Your providers, your keys, your models

- An **org-scoped credential vault** (AES-256-GCM envelope encryption, sealed by `HELIO_ENCRYPTION_KEY`) stores every secret you hand Helio. Credentials are verified against the live provider when you save them.
- **Real per-organization sending**: SMTP, Postmark, Resend, or Mailgun for email; Twilio for SMS; WhatsApp Cloud API — plus a per-org system mailer and system alerts when sending breaks.
- **Bring your own AI key**: Groq, OpenAI, Anthropic, or a local server — per organization, with env fallback for single-tenant installs. The copilot tells you plainly when no AI is connected and links you to Settings.
- **Bring your own churn model**: ONNX, XGBoost JSON, or an HTTPS endpoint, executed in a sandboxed subprocess with a safe fallback chain. Helio maps its computed contact features to your model automatically — no configuration screen at all.

### An admin control room

- A dedicated **Admin** area for owners and admins: the **audit log viewer**, **usage and sales reports**, **system health** (every service, store, and backup at a glance, with versions), and **alerts**.
- The **Database Studio**: browse and edit your own data with validation, re-authentication for writes, and a full audit trail — no third-party DB tool needed.
- A real **RBAC permission matrix** governs every surface; member, admin, and owner mean what they say.

### A CRM that closes the loop

- **Contacts and deals open as full pages** — traits, timeline, notes, linked deals, meetings.
- **Companies** as first-class records, linked from contacts and deals.
- **Notes everywhere**, a **table view for the deal pipeline** alongside the board, sales **reports**, tasks grouped by due date, and the meeting scheduler from v1 — now one connected loop.

### Switching is a wizard, not a weekend

- A **migration wizard** imports from **HubSpot, Mailchimp, or Klaviyo** with API-token connectors, or from any **CSV** with a column-mapping step that shows every column, an example value, and where it lands.

### The solar-luxe redesign

- A complete design system: warm-ivory light theme, layered-ink dark theme, gold accents, the Fraunces editorial face on every page title, a sectioned sidebar (Audience / Engage / Intelligence / Sales / System) with a gold active rail, and a denser, chart-rich dashboard.
- **The copilot became a studio**: chat-first, suggestion chips, and drafts that _show what they mean_ — readable rule summaries instead of raw JSON, editable names before anything is saved.
- **The email builder grew up**: a visual block palette, and image blocks that take **uploads from your device** (stored in Helio, served by your instance) with width, alignment, and corner-radius controls applied identically in preview and send. Upload your **organization logo** the same way.
- **Live previews everywhere**: widgets and in-app messages preview as you type, the landing editor mirrors the published page, form cards embed the real hosted form, and the campaign composer renders your actual template through the real send pipeline.
- Theme choice at login, a back-to-top control in the house style, and WCAG 2.2 AA contrast verified by axe on every audited page.

### Security, honestly

- **Forgot-password** end to end, **scoped API keys**, **org-enforced two-factor** with an enrollment flow that opens right from the requirement banner, a **password rotation policy**, and **invite-only instances by default** once setup completes.
- **Legal pages** ship in the docs: privacy, terms, and a data-and-compliance map (RLS tenant isolation, the credential vault, GDPR export/erase, RFC 8058 one-click unsubscribe, an operator checklist). Helio's only outbound call is an anonymous GitHub version check — and you can turn that off.

## Breaking changes & upgrade notes (from v1.0.0)

- **Billing is gone.** The Stripe integration, plans, and contact caps no longer exist. Nothing to configure, nothing to pay.
- **New required secret:** `HELIO_ENCRYPTION_KEY` seals provider credentials at rest. Fresh installs generate it automatically; source/compose upgrades must add it to `.env` (see `.env.example`).
- **Instances are invite-only after setup.** Public signup is off once the setup wizard completes; `ALLOW_PUBLIC_SIGNUP=true` opts back in.
- Upgrading a v1 compose deployment? Back up first, then follow the self-hosting guide's upgrade section — the `helio` CLI install is the recommended path for new machines.
