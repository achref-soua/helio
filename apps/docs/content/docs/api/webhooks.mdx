title: Webhooks

---

Helio both **sends** webhooks (from journeys) and **receives** them (from
delivery and billing providers).

## Outbound — the journey webhook node

A [journey](/docs/guides/journeys) can call an external URL with a `webhook`
node — to notify your own systems, post to chat, or hand off to another tool as
a contact reaches a step.

When `WEBHOOK_SIGNING_SECRET` is set, each outbound request carries an
`x-helio-signature` header: an HMAC-SHA256 of the raw body. Verify it on your
side before trusting the payload.

```ts
import { createHmac, timingSafeEqual } from 'node:crypto';

function verify(rawBody: string, signature: string, secret: string): boolean {
  const expected = createHmac('sha256', secret).update(rawBody).digest('hex');
  const a = Buffer.from(expected);
  const b = Buffer.from(signature);
  return a.length === b.length && timingSafeEqual(a, b);
}
```

<Callout type="warn">
  Always verify the signature against the **raw** request body, before any JSON parsing or
  re-serialization — re-serializing can change bytes and break the HMAC.
</Callout>

## Inbound — provider webhooks

| Source                 | Endpoint               | Effect                                                      |
| ---------------------- | ---------------------- | ----------------------------------------------------------- |
| Email bounce/complaint | tracking / delivery    | Adds the address to the **suppression list** automatically. |
| Stripe (hosted only)   | gateway, outside `/v1` | Maps a subscription's price to a plan; signature-verified.  |

### Stripe billing webhook

When `STRIPE_WEBHOOK_SECRET` is configured, the gateway accepts Stripe events at
a dedicated route (mounted outside `/v1`, authenticated by Stripe's signature
rather than an API key). It handles `checkout.session.completed` and
`customer.subscription.*`, upserting the organization's subscription and
deriving the plan from the price. Until configured, the route returns `404`.
Self-hosted Helio is unlimited and unmetered, so this is optional — see
[Billing configuration](/docs/configuration#billing-hosted-only-optional).

## Roadmap

A general-purpose outbound webhook subscription system (subscribe to contact,
campaign, and journey events with delivery retries) is on the
[roadmap](https://github.com/achref-soua/helio#roadmap). Today's outbound path
is the journey `webhook` node.
