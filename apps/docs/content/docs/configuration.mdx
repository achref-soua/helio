Helio reads all configuration from the environment. The authoritative,
always-current list is [`.env.example`](https://github.com/achref-soua/helio/blob/main/.env.example)
at the repository root ‚Äî every variable is documented there in the same pull
request that introduces it. This page summarizes them by area.

<Callout type="warn">
  Variables marked **required** have no default and make the owning service fail fast at startup.
  Never commit real secrets ‚Äî inject them through your environment or a secrets manager.
</Callout>

## PostgreSQL

| Variable                         | Required | Default                                           | Purpose                                                     |
| -------------------------------- | -------- | ------------------------------------------------- | ----------------------------------------------------------- |
| `DATABASE_URL`                   | ‚úÖ       | `postgresql://helio_app:‚Ä¶@localhost:5432/helio`   | Runtime connection. Uses the RLS-bound `helio_app` role.    |
| `DATABASE_ADMIN_URL`             | for DDL  | `postgresql://helio:‚Ä¶@localhost:5432/helio`       | Schema owner, bypasses RLS ‚Äî used only by migrate and seed. |
| `POSTGRES_USER/PASSWORD/DB/PORT` | ‚Äî        | `helio` / `helio_dev_password` / `helio` / `5432` | Local container credentials.                                |

The split between `DATABASE_URL` (tenant-scoped, row-level security enforced)
and `DATABASE_ADMIN_URL` (schema owner) is a core part of the
[tenancy model](/docs/concepts/tenancy).

### AI security

Keys are read as secrets and never logged. Prompts are refused over plaintext
HTTP to a non-local endpoint unless you explicitly opt out. Copilot retrieval
runs over the RLS-bound application role, never the admin connection ‚Äî so the
model can never reach another tenant's data. For full data sovereignty, point
`INTEL_LLM_PROVIDER` at `ollama` or `local`.

## Release identity & updates

Release images bake the version and commit in at build time; every service
reports them on `/healthz`, and the dashboard shows them in the sidebar and
under **Settings ‚Üí About**. Source checkouts report `dev`.

| Variable             | Required | Purpose                                                                                                                            |
| -------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| `HELIO_VERSION`      | ‚Äî        | Release version (set by image build args; leave unset in dev).                                                                     |
| `HELIO_COMMIT`       | ‚Äî        | Build commit hash (set by image build args).                                                                                       |
| `HELIO_UPDATE_CHECK` | ‚Äî        | The dashboard polls GitHub's public releases feed (cached, anonymous, sends nothing) and shows an update notice. `false` disables. |
