**Admin** (in the sidebar, visible to admins and owners) is the
organization's control room. It opens on the **audit log**; reports and
system health land alongside it.

## Audit log

Every security-relevant action in Helio writes an audit entry: sign-ins,
two-factor changes, password changes, member invitations and role changes,
every settings change (credentials, SSO/SCIM, API keys, webhooks,
integrations, domains, branding, churn models), content lifecycle events
(campaign and journey launches, imports and exports), backup requests, and
each analytics SQL execution — including attempts made while the analytics
store was down.

The viewer shows the trail newest-first with:

- **Action starts with** — filter by action family, e.g. `auth.` for
  sign-ins and credential changes, `member.` for invitations and role
  changes, `campaign.` for launches.
- **Actor** — match by member name or email.
- **From / To** — a date window.
- **Export CSV** — the current filter as a file, capped at 1,000 rows (a
  notice tells you when the cap cut the result; narrow the window for the
  rest).

Entries name the actor (or `system` for unattended writes), the action,
and the target object. Rows written by the auth kernel (sign-ins, member
changes) appear even when they happened outside the dashboard — SCIM
provisioning included.

## Reports

**Admin → Reports** answers the organization-wide questions in one place:

- **Messages sent** — email and in-app deliveries per day (30/60/90-day
  window).
- **Contact growth** — new contacts per day.
- **Top campaigns** — by emails sent, with opens and clicks when the
  analytics store is reachable; when it isn't, the card says so plainly
  and the send counts stay accurate.
- **Journey outcomes** — enrollments per journey by status.
- **Member activity** — audited actions per member over the last 30 days.

Tabular cards download as CSV. Numbers come from Helio's own database
first, so the page works on the core profile without ClickHouse.

## System health

**Admin → Health** is the page to open when something feels off:

- **Services** — every Helio service with the version it reports, checked
  live. A core-profile install shows the full-profile services as _Off_,
  not as errors.
- **Stores & checks** — Postgres, Redis, ClickHouse, and Temporal
  reachability; the age of the last successful backup (flagged after 36
  hours); failed provider credentials and churn models that need
  attention.
- **System alerts** — the feed that send, backup, and model failures
  raise (one unread alert per cause). The bell in the top bar shows the
  unread count everywhere in the app; mark alerts read here or from the
  bell.

<Callout type="info">
  Who can see this? `admin:audit` / `admin:reports` / `admin:health` — admins and owners. The full
  role matrix lives in [Roles & permissions](/docs/security/permissions).
</Callout>
