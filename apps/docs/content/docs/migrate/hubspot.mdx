---
title: Migrate from HubSpot
description: Pull your contacts and companies straight from HubSpot with a private-app token — or walk the CSV wizard with a HubSpot export.
---

Two ways in; both end in the same guided import.

## One-click: connect HubSpot

1. In HubSpot, create a **private app** (Settings → Integrations → Private
   apps) with the `crm.objects.contacts.read` scope and copy its access
   token.
2. In Helio, **Settings → Provider credentials → Add** → kind **HubSpot**,
   paste the token. It is sealed into the credential vault and never shown
   again.
3. **Contacts → Import CSV → Or pull straight from a platform** — pick the
   connection and **Pull contacts**.

Helio pages through your contacts (email, first/last name, company,
opt-out flag), then runs the same background import as the CSV path:

- **Companies** come along — matched by name per workspace or created.
- **Opt-outs are honored**: a HubSpot `hs_email_optout` contact lands
  UNSUBSCRIBED and is never mailed by any Helio sender.
- Progress is live; rejected rows download as a CSV with reasons.

<Callout type="info">
  Deals are not pulled automatically — stage mapping between arbitrary HubSpot pipelines and yours
  is a judgment call, not a guess Helio should make. Export deals to CSV in HubSpot and create them
  on the Deals board, or via the REST API.
</Callout>

## CSV fallback

Export contacts from HubSpot (Contacts → Export) and drop the file on the
import wizard — HubSpot exports are detected automatically, and the
column-mapping step lets you adjust anything before a row is written.
